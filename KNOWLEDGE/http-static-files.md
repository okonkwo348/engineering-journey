# Go HTTP Static Files

## What are Static Files?

Static files are resources that the server sends to the browser exactly as they exist without modifying their contents.

Examples include:

* CSS
* JavaScript
* Images
* Fonts
* Videos
* Icons

---

## Dynamic vs Static Content

### Dynamic Content

Generated by Go before being sent to the browser.

Examples:

* HTML templates
* ASCII Art output
* Form responses

---

### Static Content

Sent directly from disk to the browser.

Examples:

* `style.css`
* `logo.png`
* `script.js`

---

## `template.ParseFiles()`

Purpose:

Reads HTML template files and prepares them for rendering.

It does **not** serve CSS or images.

Example:

```go
ts := template.Must(template.ParseFiles("templates/index.html"))
```

---

## `http.FileServer`

Purpose:

Returns an HTTP handler that serves static files from a directory.

Example:

```go
http.FileServer(http.Dir("static"))
```

---

## `http.Dir`

Purpose:

Defines the root directory that the file server will use when searching for files.

Example:

```go
http.Dir("static")
```

This tells the file server:

> Start looking inside the `static` folder.

---

## `http.StripPrefix`

Purpose:

Removes a prefix from the incoming URL before passing the request to the file server.

Example:

Browser requests:

```
/static/css/style.css
```

After stripping:

```
css/style.css
```

The file server then locates:

```
static/css/style.css
```

---

## Complete Request Flow

1. Browser requests `/`.
2. HomeHandler renders `index.html`.
3. Browser finds:

```html
<link rel="stylesheet" href="/static/css/style.css">
```

4. Browser automatically sends:

```
GET /static/css/style.css
```

5. Go router matches `/static/`.
6. `http.StripPrefix()` removes `/static/`.
7. `http.FileServer()` searches inside the `static` directory.
8. The CSS file is returned.
9. The browser applies the styles.

---

## Engineering Principle

Static assets belong in the presentation layer.

Changes to CSS should not require changes to:

* Business Logic
* Handlers
* Banner files

This is an application of the Separation of Concerns principle.

---

## Key Takeaways

* `template.ParseFiles()` renders HTML templates.
* `http.FileServer()` serves static files.
* `http.Dir()` defines the file server's root directory.
* `http.StripPrefix()` adjusts request paths so the file server can locate files.
* Browsers automatically request linked CSS files after loading an HTML page.
